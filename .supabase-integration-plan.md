Supabase integration plan for REVT

1. Add environment variables
   - VITE_SUPABASE_URL
   - VITE_SUPABASE_ANON_KEY
   - (Server only) SUPABASE_SERVICE_ROLE_KEY

2. Client-side auth
   - Create `src/auth/AuthProvider.tsx` React context to provide user/session
   - Create `src/hooks/useUser.ts` hook
   - Update `client/pages/Auth.tsx` to call supabase auth APIs (signIn/signUp/magic link)
   - Add `ProtectedRoute` wrapper for routes needing auth

3. Storage helpers
   - Create `utils/supabaseStorage.ts` with helpers: uploadFile, getPublicUrl, createSignedUrl
   - Use storage bucket `invoices` with folder by org (e.g., `org-id/invoices/...`)

4. Server-side secure actions (optional but recommended)
   - Add server endpoint (Express `server/routes/supabase.ts`) that uses `SUPABASE_SERVICE_ROLE_KEY` to create signed URLs for private files or to upload files server-side

5. Database & RLS (optional)
   - Add `profiles` table linking `user_id` to `org_id` and enforce RLS policies

6. Testing & verification
   - Start dev server, create account, upload/download sample invoice PDF

Notes:
- Never expose service role key in client-side code.
- Use signed URLs for private file access.
